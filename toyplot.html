Axes2D.bars(a, b=None, c=None, along='x', baseline='stacked', fill=None, palette=None, opacity=1.0, width=0.8, title=None, style={}, id=None)[source]¶

Add traditional vertical bar(s) to the axes.

Parameters:

a, b, c : array-like sets of coordinates

If a, b, and c are provided, they specify the bar X coordinates, bottom coordinates, and top coordinates respectively. If only a and b are provided, they specify the bar X coordinates and top coordinates, with the bottom of each bar on the origin. If only a is provided, it either specifies the top of each bar, with the bottoms on the origin and the X coordinates ranging from [0, N), or a tuple containing bar heights and bar edges. Note that this latter behavior allows you to pass the results from numpy.histogram() directly to this function.

fill : array-like set of colors, optional

Specify a single color for all the bars, one color per bar, or a per-bar scalar value to be mapped to a color using the palette parameter.

palette : toyplot.color.Palette, optional

Palette to be used for mapping scalar fill values to colors. If unspecified, a default palette is used.

opacity : array-like set of opacities, optional

Specify a single opacity for all the bars, or one opacity per bar.

width : array-like set of bar widths, optional

Specify a single width or collection of bar widths. Note: when grouping bars with toyplot.BarMagnitudesMark.group(), this value will control the width of each group of bars.

style : dict, optional

Collection of CSS styles to apply to the mark. See toyplot.BarMagnitudesMark for a list of useful styles.

id : string, optional
Returns:

bars : toyplot.BarMagnitudesMark

Axes2D.legend(marks, bounds=None, rect=None, corner=None, grid=None, gutter=50, style={}, label_style={}, id=None)[source]¶

Add a legend to the axes.

Parameters:

marks : sequence of marks to add to the legend

Each mark to be displayed in the legend should be specified using either a (label, mark) tuple or a (label, mark, style) tuple. Each label should be the human-readable text to be displayed next to the mark. The mark can be a string value “line” or “rect”, a marker string “o”, “s”, “^”, or an actual intance of toyplot.Mark.

bounds : (xmin, xmax, ymin, ymax) tuple, optional

Use the bounds property to position / size the legend by specifying the position of each of its boundaries. The boundaries may be specified in absolute drawing units, or as a percentage of the axes width / height using strings that end with “%”.

rect : (x, y, width, height) tuple, optional

Use the rect property to position / size the legend by specifying its upper-left-hand corner, width, and height. Each parameter may be specified in absolute drawing units, or as a percentage of the axes width / height using strings that end with “%”.

corner : (corner, width, height, inset) tuple, optional

Use the corner property to position / size the legend by specifying its width and height, plus an inset from a corner of the axes. Allowed corner values are “top-left”, “top”, “top-right”, “right”, “bottom-right”, “bottom”, “bottom-left”, and “left”. The width and height may be specified in absolute drawing units, or as a percentage of the axes width / height using strings that end with “%”. The inset is specified in absolute drawing units.

grid : (rows, columns, index) tuple, or (rows, columns, i, j) tuple, or (rows, columns, i, rowspan, j, columnspan) tuple, optional

Use the grid property to position / size the legend using a collection of grid cells filling the axes. Specify the number of rows and columns in the grid, then specify either a zero-based cell index (which runs in left-ot-right, top-to-bottom order), a pair of i, j cell coordinates, or a set of i, column-span, j, row-span coordinates so the legend can cover more than one cell.

gutter : size of the gutter around grid cells, optional

Specifies the amount of empty space to leave between grid cells When using the grid parameter to position the legend.

style : dict, optional

id : string, optional
Returns:

legend : toyplot.LegendMark

Bases: object

Top-level container for Toyplot drawings.

Parameters:

width : integer, optional

Width of the canvas in drawing units.

height : integer, optional

Height of the canvas in drawing units.

style : dict, optional

Collection of CSS styles to apply to the canvas.

autorender : string or False, optional

Specify the name of a backend to use for autorendering or False to disable. Valid backend names are “html”, “png”, or “svg”.

Examples

The following would create a Canvas 800 units wide, 600 units tall, with a yellow background:

>>> canvas = toyplot.Canvas(800, 600, style={"background-color":"yellow"})

Methods

animate(frames, callback=None)[source]¶

Generate a collection of animation frames, calling a callback to store an explicit representation of what changes at each frame.

Parameters:

frames : integer, tuple, or sequence

Pass a sequence of values that specify the time (in seconds) of the beginning / end of each frame. Note that the number of frames will be the length of the sequence minus one. Alternatively, you can pass a 2-tuple containing the number of frames and the frame rate in frames-per-second. Finally, you may simply specify the number of frames, in which case the rate will default to 30 frames-per-second.

callback : function

The callback function will be called once per frame, and will receive an instance of toyplot.AnimationFrame as its sole argument. The callback function can access the frame number, time, and duration from the state object, and should use the other methods provided by the state object to make changes to the canvas.
autorender(method)[source]¶

Enable / disable canvas autorendering.

Autorendering - which is enabled by default when a canvas is created - controls how the canvas should be displayed automatically without caller intervention in certain interactive environments, such as IPython notebooks.

Note that autorendering is disabled when a canvas is explicitly shown using any of the rendering backends.

Parameters:

method : string or False

Specify the name of a backend to use for autorendering or False to disable. Valid backend names are “html”, “png”, or “svg”.
